The challenge

A marketplace-style platform wants to know where it's losing customers on the path from visit to purchase, and what a first-month customer is actually worth over time — questions that raw event logs don't answer on their own.

The approach

  • Queried raw funnel events (visit, cart, purchase) in SQL and structured them into a clean, stage-by-stage table.
  • Used Python (pandas) to calculate drop-off rates at each funnel stage and build month-1 cohort retention curves.
  • Estimated LTV for month-1 cohorts based on their subsequent purchase behavior.
  • Exported the clean tables into Power BI to build a funnel and cohort visualization layer.

The result

The funnel analysis pinpointed exactly which stage lost the most customers, and the cohort view showed how much a typical month-1 customer was worth over their first few months — turning a vague growth question into two specific, actionable numbers.

  • Funnel drop-off quantified at each stage
  • Month-1 cohort retention curve built in Python
  • Python analysis exported cleanly into a Power BI presentation layer

What I'd do differently

Bridging the Python analysis into Power BI cleanly took more iteration than I expected — I initially exported far more columns than the dashboard needed. Next time, I'd design the handoff format between the two tools before starting the analysis, not after.
