The challenge

Every company has some daily or nightly data pull that, without automation, quietly depends on someone remembering to run it — and on nobody being on vacation the day it breaks.

The approach

  • Connected to an external API (exchange rates or weather data) to pull fresh data on a schedule.
  • Wrote the ingestion logic in Python, with basic logging and error handling so failures are visible instead of silent.
  • Scheduled the pipeline with Airflow to run automatically, without manual triggering.
  • Loaded the ingested data into a Snowflake warehouse, ready for downstream modeling.

The result

The pipeline now runs unattended on schedule, with failures logged instead of discovered days later when someone notices stale data.

  • Fully automated daily ingestion
  • Basic error handling and logging in place
  • Feeds directly into the warehouse, ready for dbt modeling

What I'd do differently

I kept the error handling fairly basic — it logs failures but doesn't yet retry or alert anyone. Adding retries and a simple alert (even just an email) is the natural next step before I'd trust this pipeline with something business-critical.
