The challenge

A retail chain (or bank branch network) needs a way to compare performance across regions — Casablanca, Rabat, Marrakech, Tangier — but the raw transactional data lives in flat exports with no shared structure, making it hard to answer a simple question: which region is underperforming, and why?

The approach

  • Designed a star schema in SQL (fact_sales, dim_branch, dim_product, dim_date) to model the transactional data.
  • Loaded and validated the data, checking for consistency across branches and time periods.
  • Connected the model to Power BI and wrote DAX measures for YoY growth and regional performance.
  • Built a branch-manager-facing dashboard with drill-through, so a manager can go from network view down to a single branch.

The result

The dashboard surfaces regional and branch-level performance at a glance, with YoY growth built into every view instead of being recalculated by hand each month.

  • Star schema with 4 dimension tables
  • DAX-driven YoY growth on every branch view
  • Drill-through from network view to single-branch level

What I'd do differently

This was my first time designing a star schema from scratch rather than working from one that already existed. Getting the grain of fact_sales right — one row per transaction line, not per order — took a couple of false starts before the DAX measures started behaving the way I expected.
